# Builder / Fluent API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `Builder` type to `dataframe` that enables fluent method chaining with short-circuit error handling.

**Architecture:** A `Builder` struct holds `(DataFrame, error)`. Every method checks `b.err != nil` first — if set, it's a no-op. Entry point is `df.Build()`, terminal is `b.Result() (DataFrame, error)`. All methods are thin delegation wrappers.

**Tech Stack:** Go, standard library only (no new dependencies)

package dataframe

// Builder enables fluent method chaining on DataFrame operations.
// It accumulates the first error encountered and short-circuits subsequent calls.
type Builder struct {
	df  DataFrame
	err error
}

// Build returns a new Builder wrapping this DataFrame.
func (df DataFrame) Build() *Builder {
	return &Builder{df: df}
}

// Result returns the final DataFrame and the first error encountered (if any).
func (b *Builder) Result() (DataFrame, error) {
	return b.df, b.err
}
```

### Builder / Fluent API

Chain multiple DataFrame operations without intermediate error handling:

```go
import (
    "fmt"
    "os"

    csvio "github.com/vchitepu/gopandas/lib/dataio/csv"
)

func main() {
    f, _ := os.Open("employees.csv")
    defer f.Close()

    df, _ := csvio.FromCSV(f)

    result, err := df.Build().
        Select("name", "age", "salary").
        Query("age >= 30").
        SortBy([]string{"salary"}, []bool{false}).
        Head(5).
        Result()
    if err != nil {
        panic(err) // first error in the chain
    }
    fmt.Println(result)
}
```

The Builder short-circuits on the first error. All subsequent calls become no-ops, and `Result()` returns that error.
